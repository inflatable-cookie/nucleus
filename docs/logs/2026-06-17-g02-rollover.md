# 2026-06-17 g02 Rollover

Status: active
Owner: Tom
Updated: 2026-06-17

## Decision

Roll from `g01` to `g02`.

## Rationale

`g01` completed a substantial foundation run:

- Northstar and Effigy spine
- Rust workspace shape
- harness, SCM, native harness, planning, memory, research, and Effigy
  contracts
- engine-first host authority correction
- local state, control DTO, command evidence, command diagnostics, and runtime
  readiness proof surfaces

The reassessment found that continuing implementation inside `g01` would keep
extending proof surfaces without resolving the central architecture question:
how Nucleus models work across tasks, sessions, harness runtimes, SCM
operations, checkpoints, hosts, and clients.

`g02` therefore starts as an orchestration-and-engine-boundary generation.

Generation size alone is not the rollover rule. The rollover is justified
because the development mode is changing from foundation/proof surfaces to the
core execution model.

## Consequence

`g01` is closed as foundation history.

`g02` is active, but paused at a planning gate until the orchestration model is
chosen and promoted into contracts.

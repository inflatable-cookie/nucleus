# g05 Product Consolidation

Status: active
Owner: Tom
Updated: 2026-07-20

## Purpose

Turn the proven product features into one coherent application. Work from the
app shell inward, removing global or proof-shaped behavior that breaks project
continuity without expanding the visible UI.

## Generation Runway

Current generation goal:

- make project switching restore a complete, predictable working context while
  keeping the normal path visually sparse

Runway bands:

- project-scoped workspace composition and minimal new-project defaults
- shell chrome, project switching, panel recovery, and empty-state cohesion
- shared Goal, Task, selection, and agent-conversation context
- editor, Diff, review, and rework workflow consolidation
- Terminal, Browser, resource targeting, and host-status consolidation
- Memory, settings, provider selection, and advanced-control placement
- accessibility, responsiveness, error-state, and end-to-end workflow passes

The operator sets the shape of each inward step after using the previous one.
Do not compile speculative specialist-panel redesigns ahead of that feedback.

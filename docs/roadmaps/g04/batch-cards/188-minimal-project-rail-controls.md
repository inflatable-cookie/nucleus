## Objective

Add name-only project creation and compact lifecycle controls to the existing
project rail.

## Acceptance

- New project asks only for a name
- rename, park, archive, restore, and delete live behind project menus
- selection and workspace state respond coherently to lifecycle changes
- loading, conflict, and refusal states are visible without modal churn

## Stop Conditions

- a project setup wizard enters the primary flow
- advanced resource or sync controls become permanent rail chrome
